3. App Permissions Explained

The following Android permissions are declared in the App and here is why each is needed:

  • INTERNET – Required to validate Pro license keys via Firebase, load ads (AdMob), check for app updates, and fetch remote configuration.
  • ACCESS_NETWORK_STATE – Checks if internet is available before making network requests, to show an offline screen instead of crashing.
  • FOREGROUND_SERVICE – Allows the app to run background optimization tasks (e.g., Force Stop Engine) as a visible foreground service so Android does not kill it.
  • POST_NOTIFICATIONS – Used to show a persistent notification when a foreground service is running, as required by Android 13+.
  • SYSTEM_ALERT_WINDOW – Required to display the floating FPS overlay on top of your running game screen.
  • VIBRATE – Used for haptic feedback on certain UI interactions within the app.
  • WAKE_LOCK – Prevents the device from sleeping while performance optimization tasks are actively running.
  • com.google.android.gms.permission.AD_ID – Required by Google AdMob to serve and measure ads using the Advertising ID. You can reset or opt out of ad personalization in Android Settings → Privacy → Ads.
  • moe.shizuku.manager.permission.API_V23 – Required to communicate with the Shizuku service, which grants the App elevated shell access without requiring root. Shizuku is a separate app you install and control.

4. Shizuku Usage

TurboFPS Pro uses Shizuku, a third-party tool that enables privileged shell commands on non-rooted Android devices. All shell commands executed via Shizuku operate locally on your device only.

The App uses Shizuku to:

  • Apply performance tweaks (CPU governor, GPU settings, thermal control, display refresh rate)
  • Install optimization modules (ZIP packages executed via local shell)
  • Write game configuration files (BGMI Active.sav) to the game's data directory
  • Force-stop background apps to free up RAM
  • Execute ADB shell commands typed by the user in the Turbo Shell screen

None of the data accessed or modified via Shizuku is transmitted to any server. All operations are performed entirely on your local device.

6. Pro License System

TurboFPS Pro includes a license key validation system. When you enter a Pro key, the App sends only the key string itself to Firebase Realtime Database to verify its validity.

No username, email, device ID, or other personal information is stored alongside license keys. Keys are validated in read-only mode — the App does not write any user data to Firebase.
